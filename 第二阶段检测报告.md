## 二、核心架构详解

### 2.1 Function Calling 工作流程

```
用户输入："我想找一位心理咨询师，价格不要太贵"
    │
    ▼
AiSearchController（接收请求）
    │
    ▼
AiSearchService.searchStream()
    │
    ▼
ChatClient.prompt()
    .system(SYSTEM_PROMPT)          ← 定义 AI 角色和行为规范
    .user(query)                     ← 传入用户自然语言
    .functions("searchSkillsByIntent") ← 注册工具函数
    .stream()                        ← 流式调用
    │
    ▼
AI 模型解析意图 → 自动调用 searchSkillsByIntent
    │  参数：{ category: "心理咨询", minPrice: 0 }
    │
    ▼
AiFunctionConfig.searchSkillsByIntent()
    │  → SkillService.searchSkills("心理咨询", null, 0.0)
    │  → 返回匹配的技能列表
    │
    ▼
AI 模型根据搜索结果生成推荐语
    │  输出：{ "recommendationText": "...", "skillIds": [1, 6] }
    │
    ▼
SSE 流式推送给前端（逐字打字机效果）
```

### 2.2 System Prompt 设计

System Prompt 是 AI 行为的核心控制文件，包含四个关键部分：

1. **角色定义**：OPC 技能交易平台的智能导购助手
2. **工作流程**：分析意图 → 调用工具 → 生成推荐
3. **推荐语规范**：温暖亲切、突出亮点、对比差异、100-200字
4. **输出格式**：严格 JSON（recommendationText + skillIds）

### 2.3 工具函数注册方式

使用 `@Bean` + `@Description` 注解方式注册：

```java
@Bean
@Description("根据用户的意图搜索技能/服务...")
public Function<SearchSkillsRequest, SearchSkillsResponse> searchSkillsByIntent() {
    return request -> { ... };
}
```

请求参数使用 record + `@JsonPropertyDescription`，AI 会根据描述自动映射自然语言到参数。

### 2.4 三种接口模式

| 接口 | 路径 | 返回类型 | 适用场景 |
|------|------|---------|---------|
| SSE 流式 | POST /api/ai/search-stream | SseEmitter | 前端 EventSource 接收 |
| Flux 流式 | POST /api/ai/search-flux | Flux<String> | 前端 fetch API 接收（推荐） |
| 同步 | POST /api/ai/search | String | 后台任务、测试 |

## 四、已知限制与注意事项

| 限制 | 说明 | 解决方案 |
|------|------|---------|
| API Key 必须配置 | 未配置 API Key 时启动会报错 | 在 application-dev.yml 中填入有效 Key |
| Function Calling 依赖模型能力 | 部分模型不支持 Function Calling | 推荐使用 DeepSeek / GPT-4o-mini |
| 流式输出含工具调用延迟 | AI 需先调用工具再生成推荐 | 前端可显示"AI 正在分析..."加载动画 |
| JSON 格式可能不稳定 | AI 有时返回带 markdown 标记的 JSON | 前端需做 JSON 解析容错处理 |
| SSE 超时 | 默认 120 秒超时 | 可根据 AI 响应速度调整 |
