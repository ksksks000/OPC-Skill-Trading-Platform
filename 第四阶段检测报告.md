## 二、架构设计详解

### 2.1 聊天系统架构

```
前端聊天页面
    │
    ├── 进入页面 → HTTP GET /user/chat/history → 拉取历史消息
    │
    ├── 连接 WebSocket → ws://host:8080/ws/chat/{userId}
    │   └── 实时接收对方消息
    │
    ├── 发送消息 → HTTP POST /user/chat/send
    │   └── 后端保存到数据库 + WebSocket 推送给接收方
    │
    └── 标记已读 → HTTP PUT /user/chat/read
```

### 2.2 WebSocket 端点设计

| 端点 | 路径 | 用途 |
|------|------|------|
| ChatWebSocketServer | `/ws/chat/{userId}` | 聊天消息实时推送 |
| WebSocketServer（原有） | `/ws/{sid}` | 订单通知广播（保留） |

两个端点互不干扰，共用同一个 `ServerEndpointExporter` Bean。

### 2.3 消息推送流程

```
发送方 → POST /user/chat/send
    │
    ▼
ChatMessageServiceImpl.sendMessage()
    ├── 1. 保存消息到 chat_message 表
    └── 2. ChatWebSocketServer.sendToUser(receiverId, json)
            ├── 接收方在线 → 通过 WebSocket 实时推送
            └── 接收方离线 → 消息仍在数据库，下次拉取历史记录可见
```

### 2.4 HTTP 接口清单

| 方法 | 路径 | 功能 | 认证 |
|------|------|------|------|
| POST | /user/chat/send | 发送消息 | JWT（user） |
| GET | /user/chat/history?targetId=xx | 获取历史记录 | JWT（user） |
| PUT | /user/chat/read | 标记已读 | JWT（user） |
| GET | /user/chat/unread-count | 未读消息数 | JWT（user） |

## 四、编译与测试说明

### 4.1 数据库更新

执行以下 SQL 创建 chat_message 表：

```sql
-- 在 opc_skill 数据库中执行
CREATE TABLE IF NOT EXISTS `chat_message` (
    `id` BIGINT NOT NULL AUTO_INCREMENT,
    `sender_id` BIGINT NOT NULL,
    `receiver_id` BIGINT NOT NULL,
    `content` TEXT NOT NULL,
    `type` VARCHAR(16) DEFAULT 'text',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `is_read` TINYINT DEFAULT 0,
    PRIMARY KEY (`id`),
    KEY `idx_sender_id` (`sender_id`),
    KEY `idx_receiver_id` (`receiver_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天消息表';
```

或直接重新执行完整的 `sql/opc_skill_init.sql`。

### 4.2 测试流程

1. 启动后端 SkyApplication
2. 使用两个浏览器窗口分别登录不同用户
3. 用户A打开聊天页面：`/pages/chat/index?sellerId=2&skillId=1`
4. 用户B打开聊天页面：`/pages/chat/index?sellerId=1`
5. 用户A发送消息 → 用户B实时收到
6. 用户B回复 → 用户A实时收到

## 五、已知限制

| 限制 | 说明 | 解决方案 |
|------|------|---------|
| 图片上传未完成 | 前端选择图片后使用临时路径，未上传到 OSS | 需调用 CommonController 上传接口 |
| 消息撤回未实现 | 不支持撤回已发送的消息 | 后续可增加撤回功能 |
| 群聊未实现 | 仅支持一对一聊天 | 后续可扩展群聊 |
| 离线推送未实现 | 用户离线时无法收到通知 | 可集成极光推送/FCM |
| 消息加密未实现 | 消息明文存储 | 敏感场景需加密 |
| WebSocket 重连 | 前端未实现断线重连 | 需增加心跳 + 自动重连机制 |
